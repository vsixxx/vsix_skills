# Practitioner Guides

## What Problem This Solves

This index collects respected practical guides that are especially useful for chart choice, critique, and newsroom-grade communication.

## When to Use It

Use this when the user needs concise examples, visual vocabularies, or a practical bridge between theory and implementation.

## Key Takeaways

- Practitioner guides are strongest when used for examples, anti-patterns, and communication heuristics.
- They complement but do not replace primary API docs or theory sources.
- Editorial and newsroom sources are strongest as process models: find the story, sketch alternatives, annotate evidence, critique, and adapt for responsive publication.
- Ambitious interactive graphics often use animation, photography, generated imagery, 3D, maps, and illustration as evidence-bearing surfaces. Treat those as art direction problems with data-bound overlays, not as decorative theming.
- Scrollytelling practitioner guidance is most useful for deciding when scroll helps, how to preserve native browser behavior, how to plan mobile fallbacks, and how to avoid decorative parallax or scrolljacking.
- Mobile visualization practitioner guidance is useful for deciding when to create sibling mobile layouts, control chart height, reduce labels, preserve the focal chart above controls, and design touch-first interactions instead of squeezing a desktop composition.

## Common Mistakes

- Treating a visual vocabulary as a rigid taxonomy.
- Using example charts without checking whether the data and task truly match.
- Copying a publication's visual identity instead of learning from its principles.
