# Theory Books and Papers

## What Problem This Solves

This index points to the highest-value theory sources for deeper reading and citation.

## When to Use It

Use this when a skill needs deeper grounding or when you want a credible source to justify a recommendation.

## Key Takeaways

- Tufte remains foundational for evidence density, integration of text and graphics, and critique of chartjunk.
- Munzner provides a rigorous design process that connects domain problem, data abstraction, encoding, and algorithm.
- Shneiderman remains foundational for interaction framing: overview first, zoom and filter, then details on demand.
- Cairo, Few, Ware, and related authors are practical bridges between theory and professional communication.
- Bostock, Ogievetsky, and Heer are foundational for data-bound, representation-transparent web visualization with D3.
- Graph drawing research is the primary theory base for auto-arranging line-connected nodes, especially layered diagrams, tidy trees, orthogonal routing, overlap removal, and readability tradeoffs.
- For graph layout, edge crossings usually matter more to human understanding than secondary aesthetics such as bend count or symmetry.
- Christiansen's explanatory-science practice is a useful bridge for stories where illustration, annotation, and data must work as one explanation.
- Narrative visualization research helps balance author-led story flow against reader-driven exploration, which is central when deciding whether scrollytelling, a stepper, small multiples, or static frames are appropriate.
- Scrollytelling research provides a practical vocabulary for graphic sequences, animated transitions, pan/zoom, moviescrollers, and show-and-play techniques.
- Mobile responsive visualization research reinforces that responsive charts should adapt visual representation, interaction, and task support to device constraints rather than only scaling a desktop view.

## Common Mistakes

- Confusing historical importance with universal correctness.
- Treating books as implementation docs rather than reasoning frameworks.
