# Theory and Principles

## What Problem This Solves

This reference gives the plugin a common design vocabulary so different skills reason from the same foundations instead of reinventing chart advice ad hoc.

## When to Use It

Use this when choosing a chart, critiquing a visual, explaining why a view works, or deciding whether to simplify, annotate, facet, or restructure a figure.

## Key Takeaways

- Tufte-style guidance favors evidence-rich visuals, direct integration of words and graphics, and restraint toward decoration.
- Bertin-style thinking reminds you that position, size, shape, hue, value, texture, orientation, and motion are not interchangeable.
- Cleveland and McGill's ordering of visual encodings is a practical reason to favor bars, dots, and aligned scales over area- and angle-heavy alternatives.
- Tukey-style exploration means plotting early and preserving surprising structure long enough to understand it before collapsing it into summary.
- Munzner's nested model helps separate domain task, data abstraction, encoding design, and implementation choice.
- Shneiderman's mantra is still a strong interaction default for analytical surfaces: overview first, zoom and filter, then details on demand.

## Common Mistakes

- Treating famous names as slogans rather than working decision frameworks.
- Using "minimalism" to justify removing necessary labels, context, or explanations.
- Copying a chart form from another domain without checking whether the task and comparison structure match.
