# Task Abstraction and Chart Selection

## What Problem This Solves

This reference helps decide what to show before debating tools or styling.

## When to Use It

Use this when the user asks "what chart should I use?" or when a visualization feels wrong but the rendering stack is not obviously the issue.

## Key Takeaways

- Start from the question: compare, rank, trend, distribute, relate, monitor, map, or explain.
- Classify the data shape: table, time series, matrix, hierarchy, graph, geography, or stream.
- Pick the smallest truthful visual form that makes the key comparison easiest.
- Use small multiples when repetition improves comparison better than stacking, color, or interaction.
- Prefer declarative grammars for standard tabular charts before bespoke renderers.

## Common Mistakes

- Choosing by chart popularity instead of task fit.
- Using maps for non-spatial comparisons.
- Picking dashboards when the real need is a focused report or figure.
