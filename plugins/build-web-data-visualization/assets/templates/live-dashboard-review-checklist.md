# Live Dashboard Review Checklist

- Screen optimizes for monitoring, not chart variety.
- Highest-priority metrics are stable in location and encoding.
- Missing, delayed, or stale data is visible.
- Live, stale, offline, partial, reconnecting, and error states are visually distinct.
- Mobile portrait shows the live state before secondary settings or diagnostics.
- Mobile landscape is available when a wide timeline, map, trace, route, or table materially improves monitoring.
- Touch targets, tap inspection, step-through controls, and reset paths work without hover.
- On-screen keyboard flows for filters, notes, or search do not hide the only critical action.
- Spotty connection behavior keeps the last known good visualization visible.
- Vibration, notifications, or alerts are user-requested, meaningful, and backed by visible in-app state.
- Interactions support reset and preserve orientation.
- Performance budgets and degraded modes are defined.
- Mobile battery, thermal, bandwidth, and low-power degradation modes are defined.
- Export or incident-report path is available for key states.
