## Human Review

- What does the first frame say?
- What does imagery or motion add that data marks alone cannot?
- What would be removed if the design started to feel decorative?
- Does a still screenshot preserve the takeaway?
- Does the mobile portrait screenshot preserve the takeaway?
- Does the mobile layout keep the main visualization visible before or alongside settings?
- Are touch, keyboard, stale/offline, and permission fallback paths clear?
- If this is a sensitive subject, can a reader tell what is measured, estimated, schematic, dated, and caveated?
- If this includes human harm, does the visual preserve dignity and proportion?

## Acceptance Checklist

- Title states the insight.
- Subtitle defines metric, unit, time, and caveat.
- Main evidence is visible without hover.
- Artifact mode is chosen before renderer.
- Codex image-generated concept sets are used when composition affects understanding, shown to the user with concise plan and interaction bullets, and approved or iterated before project changes or implementation code begins.
- Approved concepts are converted into binding semantic design contracts and preserve claim, evidence hierarchy, source context, caveats, labels, layout intent, visual hierarchy, interaction staging, mobile portrait, optional landscape, and export continuation.
- Final results match the approved concept contract or document approved meaning-preserving deviations.
- Imagery or illustration carries analytical meaning if used.
- Animation has an explanatory verb if used.
- Labels are direct or immediately adjacent.
- Color is sparse and intentional.
- Annotations explain significance.
- Mobile portrait layout has been designed, not squeezed, and mobile landscape is designed when the evidence or interaction needs it.
- Accessibility summary, source, and notes are included.
- Sensitive stories include dated evidence states, source hierarchy, caveats, and humane framing.
