# Investment Banking User Context Config

Plugin authors edit this file when adding, removing, renaming, or clarifying the starter categories copied into a user's `user-context.md`. The initializer adds `## Saved Links And Context` with `status: not provided` beneath each category.

## Default Categories

# Banker Profile And Deliverable Style

- Description: Coverage group, audience defaults, approved templates, and preferred memo, deck, or workbook conventions that should shape future Investment Banking work.

# Mandate And Process Conventions

- Description: Reusable process stages, approval expectations, escalation paths, naming conventions, and review cadence for banker-owned workflows.

# Trusted Deal Sources And VDR Pointers

- Description: Trusted source-of-truth locations such as Datasite projects, Drive folders, VDR export locations, process-letter repositories, diligence folders, and live tracker pointers.

# Model Workbook And Valuation Conventions

- Description: Approved workbook templates, tab conventions, house comps guidance, valuation preferences, output expectations, and model-review rules.

# Coverage Relationship And Market Data Preferences

- Description: Relationship-source pointers, preferred first-look providers, accepted manual exports, and reusable source-priority guidance.

# Confidentiality And Review Rules

- Description: Circulation restrictions, redaction expectations, senior-review gates, and client-facing boundaries for Investment Banking artifacts.
